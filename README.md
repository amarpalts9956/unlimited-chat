# Memory Orbit (Offline-First Local Folder Web App)

Memory Orbit is a production-ready web application that provides a rich calendar + chat-like memory interface, while storing all user data only inside a folder selected by the user on their own local computer.

No cloud database is used.

## Core Idea

- The app runs in the browser.
- User selects a local folder using File System Access API.
- App creates/uses app-specific files in that folder.
- Every memory message and image is saved in that folder only.
- Re-selecting the same folder later restores the full history instantly.

## Main Features Implemented

- 2x2 matrix layout:
  - Top-left: advanced calendar + real-time date/time clock
  - Bottom-left: chat composer (text + image upload + emoji picker)
  - Right side (spanning both rows): instant timeline for selected date
- Calendar with month/year switching and date selection
- Instant load of existing memories for clicked date
- Inline editing of saved text/image captions directly in timeline
- Folder onboarding:
  - first use with empty folder initializes app files
  - repeat use with same folder detects and loads existing data
- No database dependency

## Storage Algorithm (High-Speed Local Retrieval)

The app uses an indexed date-based layout for fast loading:

1. On folder selection:
	- Check/create root metadata file `.memory-orbit.json`
	- Validate `signature` to detect valid app folder
2. Ensure app data directories exist:
	- `memory-orbit-data/entries`
	- `memory-orbit-data/assets`
3. Load lightweight index file once:
	- `memory-orbit-data/date-index.json`
4. On date click:
	- Read only that date file: `entries/YYYY-MM-DD.json`
	- Render immediately (lazy per-date loading)
5. On save:
	- Append new text/image entry to selected date file
	- Save images under `assets/`
	- Update `date-index.json`
6. On same folder reopen:
	- Detect valid metadata signature
	- Load index quickly
	- Any selected date resolves instantly from date file

This pattern avoids scanning all memory files at startup and keeps load time fast even as data grows.

## Security Notes

- Data remains in the user-selected local folder.
- App does not send memory data to any database by default.
- Browser grants folder permissions explicitly by user action.
- Optional local passphrase encryption is supported (AES-GCM via Web Crypto).

## Encryption Mode (New)

When a user selects a folder:

1. If the folder has no encryption config yet, app asks whether to enable encryption.
2. If enabled, user creates a passphrase.
3. App derives a key using PBKDF2 + SHA-256 and stores only:
	- salt
	- iteration count
	- encrypted verifier token
4. Memory entry files for each date are saved as encrypted payloads.
5. On future opens of same folder, user must enter passphrase to unlock.

Crypto config file location:

- `memory-orbit-data/crypto.json`

Important:

- If passphrase is lost, encrypted memory data cannot be recovered.
- Keep passphrase safe.

### Change Passphrase

- Open your encrypted folder.
- Click `Change Passphrase` in the top bar.
- Enter current passphrase.
- Enter and confirm new passphrase.

The app will re-encrypt all date entry files with the new key, then update `memory-orbit-data/crypto.json` with new salt and verifier.
